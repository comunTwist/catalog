Не самый юзер-френдли интерфейс, но надеюсь этого и не требовалось. Структура слегка запутана, потому опишу.

Дамп базы данных можно найти в корне проекта (se_catalog.sql), либо необходимо запустить миграции:

php yii migrate/up --migrationPath=@vendor/costa-rico/yii2-images/migrations
php yii migrate



У сущности «Категория» есть id и slug, у сущности «Товар» помимо этих двух есть price. Все  остальные поля нужно переводить, потому они вынесены в отдельные таблицы и под них созданы отдельные модели. То есть связью many to one они подтягиваются к основным сущностям.

Сначала нужно создать категорию или товар (без перевода). Затем, перейдя в их вид, можно будет добавить перевод, перейдя по соответствующей ссылке. После добавления, эта информация будет доступна в виде. У всех переведённых сущностей также есть свой вид, но это не важно.

Фотографии отображаются при создании/редактировании товаров, а также в их отдельных видах. В этих самых отдельных видах можно оставлять комментарии. Комментарии связаны с переведённым товаром, поэтому при переключении языка, комментарии могут исчезать или появляться. К сожалению, редактировать и удалять их пока нельзя. Хотел реализовать это с помощью аякса. Сейчас сервер отдаёт 400-ю ошибку. Пока не разобрался почему.

Валидация происходит встроенным в Yii2 функционалом.

Что касается, перевода сайта, то идею реализации подсмотрел в одном блоге.
